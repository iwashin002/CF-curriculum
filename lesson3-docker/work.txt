①dockerとは・・・
アプリの開発、移動、実行するためのプラットフォームのこと。他の中身に干渉しない分離した空間である「コンテナ」を
活用することでアプリの構築、移管、運用管理が容易になりコストや手間を軽減できる。

②仮想環境の種類とそれぞれの特色・・・
1.ホストOS型「物理ハードウェア＋ホストOS＋仮想化ソフトウェア」
OS上にインストールした仮想環境ソフトウェアで仮想マシンを作る。家の中に新しい部屋を増築するイメージ。
既存のパソコンやサーバーをそのまま利用できる手軽さがメリットである。その反面、複数の仮想マシンを運用する場合は
ハードウェアリソースと仮想マシンの間にホストOSが介入するため高速な処理が行えない。

2.ハイパーバイザー型「物理ハードウェア＋ハイパーバイザー」
パソコン上にインストールした仮想環境ソフトウェアが仮想マシンを作る。土地の上に家を建てるイメージ。
ワイパーバイザーという建築家（ソフトウェア）が各家（ゲストOS）へ互いに干渉しないように管理する。
複数台の仮想マシン運用でも高速な処理が実現できる。ただし、仮想環境を構築するために専用の物理サーバーを準備
しなければならず、OSごとに入れ替えが必要になる。

3.コンテナ型「物理ハードウェア＋ホストOS＋コンテナエンジン」
コンテナと呼ばれるアプリケーションを実行するためのリソースがまとめられたパッケージ。
すでにある家の中に仕切り（コンテナ）を作る。増築ではなく、新しく仕切りを作ることで部屋を増やしていくイメージ。
メリットは可搬性の高さとシステム資源の負担が少ないことで、デメリットは仮想マシンより自由度が低いことである。
「docker」はコンテナ型に分類される。

③dockerのコマンド・・・
・コンテナ起動
docker start {コンテナ名orコンテナID}
・停止
docker stop {コンテナ名orコンテナID}
・削除
docker rm {コンテナ名orコンテナID}
